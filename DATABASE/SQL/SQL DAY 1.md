# SQL DAY 1

## 모델링(Modeling)

### 모델링이란?

현실세계의 사물 또는 개념을 일정하게 도식화 한 것을 모델(Model)이라고 한다. 또한 대부분 약속된 표기법을 따라야 하는데, 그 이유는 모델링을 하는 가장 중요한 이유 중 하나가 "업무의 흐름을 가시화 하고 명세화" 하여 효율적으로 설계하고 개발하기 위해서이기 때문이다.

즉 서로간에 약속되지 않은 모델(Model)은 결코 업무에 활용 될 수 없다.

### 모델링의 특징

1. 추상화(Abstraction)

- 모델링 하고자 하는 대상의 주요한 특징을 추출하여 일정한 형식으로 표현
- 객체지향 프로그래밍에서의 추상화도 이와 같은 개념

2. 단순화(Simpleification)

- UML(Unified Modeling Language)나 ERD(Entity RelationShip Diagram)과 같은 약속된 표기법을 통해 누구나 이해하기 쉽도록 표현

3. 명확화(Clarity)

- 모델링을 보는 사람마다 다르게 해석되지 않도록 모호함 없이 명확하게 표현

### 모델링 시 유의사항

1. 중복 최소화

- 데이터베이스 내 여러 테이블에 동일한 데이터가 중복 저장되지 않아야 한다.

2. 비유연성 최소화

- 데이터 또는 프로세스의 변화가 데이터베이스에 미치는 영향을 최소화 해야한다.

3. 비일관성 최소화

- 데이터 간의 연관관계를 명확하게 정의해야 한다. 또한 일관성이 보장되야 한다.

### 모델링의 관점

1. 데이터 관점(WHAT)

- 정적 분석
- 구조 분석

2. 프로세스 관점(HOW)

- 동적 분석
- 도메인 분석

3. 상관관계 관점(INTERACTION)

- CRUD 분석

### 모델링 단계

1. 개념적 데이터 모델링
2. 논리적 데이터 모델링
3. 물리적 데이터 모델링

## 엔티티(Entity)

### 엔티티의 개념

데이터베이스 내부에서 독립적으로 존재하는 객체(Object) 또는 개념을 말한다.

토마스 브루스는 엔티티의 정의에 대해 "정보가 저장될 수 있는 사람, 장소, 물건, 사건 그리고 개념"이라고 정의하기도 하였다.

이와같이 엔티티는 하나의 개념에 국한되지 않고 포괄적인 개념에서의 객체에 대해 설명하는 것을 알 수 있다.

즉, 도서관에 대해 모델링을 진행한다고 할때, 회원, 도서, 도서관, 대출 등등을 모두 엔티티로 정의할 수 있다.
